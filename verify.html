Document Integrity Verification

Verify Report Integrity

Enter the Report ID, timestamp, case hash, and report content to verify that a ROSS report has not been altered since generation. The SHA-256 hash is computed client-side and compared against the original document hash.

Verification performed client-side using Web Crypto API (SHA-256, FIPS 180-4).
Admissibility basis: Bharatiya Sakshya Adhiniyam 2023 §61-65 (electronic records), §65B (certificate of authenticity), IT Act 2000 §85B (presumption of integrity for secure electronic records).
Hash match confirms the document has not been altered since generation by ROSS.

How Verification Works

ROSS generates a SHA-256 hash of a canonical string combining the Report ID, timestamp, case hash, and full report text. This hash is embedded in the PDF certificate and can be independently recomputed at any time.

Any modification to the report — even a single character — produces a completely different hash, proving tampering. This is the basis for electronic record integrity under Indian law.

Canonical format:
REPORT_ID:[id]\nTIMESTAMP:[timestamp]\nCASE_HASH:[caseHash]\nCONTENT:[reportText]
